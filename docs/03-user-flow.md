## 1. Goal

Every path through the site bends toward one destination: **the visitor makes contact.** Secondary conversion is a CV download. A portfolio that only informs has failed; it must convert.

## 3. Persona paths

**Persona A — recruiter.** Lands on hero → sees "open to internships" tag → scans about and skills → downloads CV → contacts (or saves the CV and contacts later). Tends to move straight down the page.

**Persona B — freelance client.** Lands on hero → skips to projects → opens the case study most similar to their own problem → returns → contacts. Needs proof of relevant capability before converting.

## 4. Design implications

| Implication | Where it shows up |
|-------------|-------------------|
| Primary CTA must appear before any scrolling. | Hero |
| Audience signal must be immediate. | Hero eyebrow tag: "open to internships + freelance" |
| CV must be reachable from both top and bottom. | Hero + contact |
| Case studies must route back toward contact, not dead-end. | App detail pages end with a contact CTA |
| Contact must be low friction. | Three fields only; alternative direct channels offered |
| Nav must allow jumping straight to projects. | Sticky anchor nav |

## 5. Friction points to avoid

- More than three form fields — each additional field measurably loses submissions.
- Contact form as the *only* route; some visitors will only use email or LinkedIn.
- Case study pages with no onward path back to contact.
- Requiring a scroll to the very bottom to find any way to make contact.
